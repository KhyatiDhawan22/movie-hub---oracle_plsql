# Movie Hub: Blockbuster Database Management System

## 🎬 Project Overview
**Movie Hub** is a backend database solution designed to modernize the operational framework of **Blockbuster LLC**. As the industry shifts toward digital-first models, this project leverages **Oracle SQL** and **PL/SQL** to transition legacy rental processes into a structured, automated, and scalable digital environment.

## 🚀 Key Features
* **Metadata Management:** Structured storage for Movies, Directors, and Production Houses.
* **Asset Tracking:** Real-time monitoring of individual physical movie copies and their availability status.
* **Membership Automation:** System-generated unique `Member_ID` allocation for new customers.
* **Transaction Logic:** Robust tracking of movie collections and returns to ensure data integrity and accountability.

## 🛠️ Tech Stack
* **Database:** Oracle Database
* **Languages:** SQL, PL/SQL (Procedures, Triggers, Functions)
* **Modeling:** Entity-Relationship (ER) Design & Data Normalization

## 📋 Problem Statement
Blockbuster faces intense competition from digital platforms like Netflix. To sustain its loyal fan base, the company requires a shift from legacy systems to a data-driven infrastructure. This project addresses the lack of a centralized system for tracking extensive movie libraries and physical inventory, providing a reliable solution for modern business needs.

## 🏗️ Database Schema
The architecture focuses on three primary pillars:
1.  **Inventory:** Managing the "status" of every movie copy.
2.  **Members:** Handling registration and unique identifiers.
3.  **Transactions:** Recording the movement of assets between the store and members.
